# Техническое задание (MVP) — TripPlanner

## Идея
Веб‑сервис для планирования поездок: пользователь создаёт поездку, добавляет активности/расходы, собирает список вещей и видит аналитику по бюджету. Для наглядности используется внешний API прогноза погоды (Open‑Meteo) и интерактивные диаграммы (Chart.js).

## Роли
1. **Гость**
   - Просмотр публичных поездок (список/детали).
2. **Пользователь (автор)**
   - Регистрация/вход.
   - Создание, редактирование, удаление своих поездок.
   - Добавление/редактирование/удаление активностей в своей поездке.
   - Управление библиотекой предметов (что брать в поездку).
   - Добавление предметов в список вещей для конкретной поездки и отметка «упаковано».

## Use cases (MVP)
- Создать поездку (наименование, направление, даты, бюджет, публичность).
- В поездке вести активности: дата, стоимость, заметка, теги.
- Видеть сумму расходов и остаток бюджета.
- Видеть диаграммы: расходы по дням и по тегам.
- Посмотреть краткий прогноз погоды для направления (через внешний API).
- Сформировать packing‑лист и отмечать упакованные вещи.

## Схема данных (ER‑описание)
- **Destination** (направление) 1—M **Trip**
- **User** 1—M **Trip** (owner)
- **Trip** 1—M **Activity**
- **User** 1—M **Tag**
- **Activity** M—M **Tag**
- **User** 1—M **PackingItem**
- **Trip** M—M **PackingItem** через **TripPackingItem** (quantity, is_packed, note)

Минимум 3 кастомные модели соблюдён (Destination, Trip, Activity, Tag, PackingItem, TripPackingItem).
